# 数据说明
---

本目录存放 **processed（冻结口径）** 的年度外生数据 CSV，用于驱动 `cchp_physical_env` 环境进行训练/评估。

很多人第一眼会觉得这些 CSV “看起来很一般/很弱小”，担心“用这样的数据做深度学习体现不出真实水平”。这里需要澄清：

- **[这不是图像/文本那种“靠海量样本堆性能”的任务]** 本项目是 **物理约束 + 多能耦合调度** 的序列决策问题。难点主要来自：
  - 多能源耦合（电-热-冷）与设备动态（GT/BES/TES/ABS/ECH/Boiler）。
  - 价格/负荷/新能源的强随机性与非平稳性。
  - 约束（爬坡、容量、SOC、驱动温度等）导致可行域不连续。
- **[数据量并不小]** 15min 分辨率下，每年固定步数为 `35040`（`365*24*4`，删除闰日 2/29）。
  - 对深度 RL 来说，训练不是“行数=样本数”这么简单；训练过程会在全年序列上反复采样 episode（7-30 天窗口），产生大量交互轨迹。
- **[数据“简单”是刻意的研究设计]** 我们冻结了最小但充分的外生变量集合（负荷/新能源/天气/价格/税），将复杂性留给环境物理与策略学习，避免“靠数据工程技巧堆出来的提升”掩盖真实机制。

---

## 文件约定（Frozen paths）

CLI 默认读取以下路径（不要改文件名）：

- 训练集：`cchp_main_15min_2024.csv`
- 评估集：`cchp_main_15min_2025.csv`

训练/评估年份在代码中是硬约束：

- 训练年必须为 `2024`
- 评估年必须为 `2025`

---

## 冻结时间口径

时间口径由 `src/cchp_physical_env/core/data.py` 冻结：

- 时区：`Asia/Shanghai`
- 分辨率：`15min`
- 全年步数：`35040`
- 闰日处理：删除 `2/29`（即使原始数据包含闰日也会被剔除）

---

## 冻结列（schema）与含义

CSV 必须至少包含以下冻结列（列名、顺序与含义冻结）：

- `timestamp`
- 负荷/新能源（MW）：
  - `p_dem_mw`：电负荷
  - `qh_dem_mw`：热负荷
  - `qc_dem_mw`：冷负荷
  - `pv_mw`：光伏出力
  - `wt_mw`：风电出力
- 天气（工程单位按字段名约定）：
  - `t_amb_k`：环境温度（K）
  - `sp_pa`：气压（Pa）
  - `rh_pct`：相对湿度（%）
  - `wind_speed`：风速
  - `wind_direction`：风向
  - `ghi_wm2` / `dni_wm2` / `dhi_wm2`：辐照度
- 价格/税（货币单位在当前研究中不强绑定具体币种，强调相对口径一致性）：
  - `price_e`：电价
  - `price_gas`：气价
  - `carbon_tax`：碳税

说明：

- 本项目将这些列视为 **外生变量**（exogenous），智能体不能改变它们。
- 环境的设备状态（如 `soc_bes`、`gt_on`、`t_tes_hot_k` 等）不是 CSV 输入列，而是在仿真中由环境内部状态转移产生。

## 数据来源（raw -> processed）

本项目的上游原始数据统一存放在 `data/raw/`，并按来源分为三类：

- `data/raw/load/`：负荷与可再生出力的来源与留痕
  - **电负荷 `p_dem_mw`**：来自 Scientific Data 2023 工业园真实电负荷数据（Suzhou, China）。来源与下载留痕见：
    - `data/raw/load/industrial_park_scidata2023/`
    - `data/raw/load/README.md`（含 DOI/OSF 与处理口径说明）
  - **热/冷负荷 `qh_dem_mw` / `qc_dem_mw`**：来自 CityLearn 公开数据集（用于构造热/冷需求序列并缩放到园区量级），见：
    - `data/raw/load/citylearn/`
  - **风电 `wt_mw`（可选）**：若启用，会从指定风电数据源生成（当前建议来源见 `data/raw/load/README.md`）。
- `data/raw/weather/`：真实天气数据来源（当前目录结构为 `data/raw/weather/oikolab/`）
  - 对应列：`t_amb_k/sp_pa/rh_pct/wind_speed/wind_direction/ghi_wm2/dni_wm2/dhi_wm2`
- `data/raw/price/`：北京市政策/公开信息整理得到的电价、气价与碳价参考
  - 对应列：`price_e/price_gas/carbon_tax`
  - 具体条目与来源链接在：`data/raw/price/catalog.csv`

> 重要：`data/raw/` 是“可溯源留痕层”，不要求格式统一；`data/processed/` 是“实验冻结口径层”，要求严格对齐与可复现。

## 数据创造与处理规范（为什么“看起来一般”是合理的）

本仓库只存放 processed 数据。processed 的含义是：

- **冻结 schema**：只允许/只依赖上述列集合，不引入“未来信息”或与评估耦合的特征工程。
- **冻结时间索引**：强制对齐到全年 15min 索引（缺失会以规则补齐/报错）。
- **冻结缺失值处理规则**：避免每个人各自“补数据”导致实验不可比。

这些规则由 `src/cchp_physical_env/core/data.py` 实施，核心行为如下：

- **时间戳解析与时区**：
  - `timestamp` 必须可解析；无时区则 localize 到 `Asia/Shanghai`，有时区则 convert。
- **单年约束**：每个 CSV 必须只包含一个年份。
- **严格递增且不可重复**：`timestamp` 必须严格递增，且不能重复。
- **对齐到预期索引**：会 `reindex` 到全年 15min 主索引。
- **缺失值修复**：
  - 负荷/新能源列（`p_dem_mw/qh_dem_mw/qc_dem_mw/pv_mw/wt_mw`）：按时间插值 + 边界前后向填充。
  - 天气列：前向填充（ZOH）。
  - 价格/税列（`price_e/price_gas/carbon_tax`）：**一旦缺失直接报错**（不允许隐式补齐），避免规则污染研究口径。

### 引用与合规（建议写进论文/答辩材料）

- 电负荷与部分外生数据来自公开论文/数据集或政府公开政策信息。论文或报告中应在“数据集/实验设置”章节引用：
  - `data/raw/load/README.md` 中列出的 DOI/项目页（负荷来源）
  - `data/raw/price/catalog.csv`（电价/气价/碳价政策与链接）
- 若你需要对外发布数据文件：
  - 建议仅发布 **处理脚本/生成流程** 与 **来源引用**，避免直接再分发受条款约束的原始数据文件。
  - `data/processed/` 的冻结口径可公开，但是否可公开分发 CSV 需遵守上游数据的许可条款。

## 使用规范（避免“体现不出真实水平”的误解）

为了保证结论可信、可复现、可对比，使用本数据时应遵守：

- **训练/评估严格分离**：
  - 训练只用 2024；评估只用 2025。
  - 任何归一化统计量、阈值（例如规则策略使用的分位数）只能从训练集拟合（代码中已限制：`compute_training_statistics` 只允许 2024）。
- **不要在 processed 数据里加入“未来信息”**：
  - 例如把未来 24h 的需求/价格直接拼成特征列，这会让策略在评估时不再是现实可部署的调度策略。
- **不要修改处理规则来“刷分”**：
  - 如果你需要新的补齐/清洗策略，请在仓库中显式实现并记录（而不是静默改 CSV），否则实验不可比。

## 数据质量/诊断建议

推荐先运行：

```bash
uv run python -m cchp_physical_env summary
```

该命令会检查冻结 schema、一致性，并输出基本统计（范围/均值/缺失率）。这份输出是论文/报告里最直接的数据“可信度证据”。

---

## 局限性（需要诚实披露）

- 这里的外生变量集合是“最小充分集”，不会覆盖所有工业现场可用的测点。
- 价格/负荷/新能源的真实性取决于上游来源与生成流程（本仓库强调 processed 口径一致性，不在此目录内重复上游采集细节）。
- 深度学习/深度强化学习的“真实水平”不仅取决于数据规模，还取决于：
  - 奖励函数/惩罚项是否合理
  - 物理约束是否正确建模
  - 训练预算（timesteps）与随机种子稳定性
  - 市场机制（是否允许外送套利等）

因此，针对“数据弱小”的质疑，我们的回答是：**我们刻意冻结了输入数据的复杂度，把研究重点放在物理一致性与策略学习的真实权衡上；并通过严格的训练/评估分年与数据校验，保证结果可复现、可对比、可解释。**
